## Diagnosis Verdict

### 根因：Lor5D 层结构退化 + layer=4 反转效应

**根因 1：层计数几乎退化（entropy collapse）**
- Lor5D@N=52: layer_count = {3:38, 4:2}，entropy = 0.29 bits（对比 Lor2D: 2.14 bits）
- Lor5D@N=72: layer_count = {3:33, 4:7}，entropy = 0.67 bits（对比 Lor2D: 2.24 bits）
- Σ_hist 方差仅为 Lor2D 的 19–46%——统计上近似退化

**根因 2：Lor5D 的 layer=4 反而增加 logH（方向反转）**
- N=52: layer=3 → mean logH=129.1, layer=4 → mean logH=**131.2** (+2.1, 反转)
- N=72: layer=3 → mean logH=196.6, layer=4 → mean logH=**199.2** (+2.6, 反转)
- 对比 Lor2D: 完美单调递减（每增一层 logH 降 1–2 个单位）

**物理解释**：
5D Lorentzian 几何下，稀疏的因果关系产生极少的层。偶然多出一层时，并非因为"更强的时间序约束"（Prediction C 的机制），而是因为高维空间中某些元素的因果连接模式发生了微调，反而允许更多的线性扩展。本质上：
- **低维（2D/3D）**：层 ↔ 全局时间序约束，Prediction C 机制有效
- **高维（5D）**：层 ↔ 局部因果重排的指纹，与全局约束脱钩

**结论**：Lor5D 反转是 **物理性边界条件**，而非统计伪影。Prediction C 的适用域应标注为 d ≤ 4（或更保守地，layer_count entropy > 1 bit 的族/N 组合）。

### 建议
- Prediction C 正文中应加入"适用域"条件：Σ_hist 需有足够方差（≥2 个有效 level）
- Lor5D@N=52/72 的反转可作为 Prediction C 的**物理边界标记**，而非证伪
- 不建议进一步增加 reps 来"修复"Lor5D——问题不是统计功效，是物理退化
