## 1. One-Sentence Claim (Frozen)

Under the frozen CG protocol (A2, keep_ratio=0.6, zeta-scan), there exists a **confirmatory parameter window** in which coarse-graining stability (`I_cg`) is a **non-trivial global filter** that predicts *dynamic-process* rank effects beyond a local-score baseline, and this survives a red-team test removing the family-switch channel.

**Frozen confirmatory window (current):**

- `N=30, keep_ratio=0.6, gamma=0.2` (A2)
- Evidence is taken primarily from `sis_runs=128` independent replications with distinct `seed_offset`.

## 2. Definitions (What D Means Here)

### 2.1 Coarse-Grained Stability Variable

We use a working, operational global variable:

- `I_cg := - zscore(penalty_cg)` within each block of fixed `(source, n, n_cg, keep_ratio, gamma, action_mode, zeta, icg_variant)`.
- `penalty_cg` is a weighted sum of: drift + family-switch + rank-shift (exact components recorded in `cg_raw_samples.csv`).

Key variants (red-team 2.0):

- `full`: drift + switch + rank
- `no_switch`: drift + rank (excludes the centroid family-switch channel)
- `switch`: switch-only
- `rank`: rank-only
- `drift`: drift-only (known to exhibit sign flips in stratified tables; not used as frozen primary)

### 2.2 Dynamic-Process Target

We explicitly avoid using identity retention itself as the third-layer target (tautology risk). Instead:

- `delta_rank := rank_eval - rank_local_eval`
- `improve_rank := -delta_rank` (larger is better; CG penalty improves rank)

This is the third-layer “dynamic-process proxy”: does CG stability predict how a family’s rank changes when the CG penalty is introduced?

### Layer 3: Independent Gain via Block Permutation (Confirmatory Backbone)

Goal: show `I_cg` predicts `improve_rank` in a way that is not an artifact of block structure, and survives removing the family-switch channel.

Statistic (per stratum):

- For each block (fixed `(source,n,n_cg,keep_ratio,gamma,action_mode,zeta,icg_variant)`), compute Spearman(`mean_I_cg`, `improve_rank`) across families.
- Average Spearman across blocks to obtain the stratum statistic.

Null:

- Within each block, permute `mean_I_cg` across families; recompute mean Spearman.

Key outputs:

- Stratified p-table: `cg_blockperm_delta_rank_stratified_refined.csv` (v10*)
- Cross-rep summary table: `outputs_confirmatory/prediction_d_dynamic_v10_freeze_strata_g02_summary.csv`

Interpretation note:

- When `n_perm=200000`, the minimum p-resolution is `1/(n_perm+1) ≈ 5e-06`. A printed p equal to that is a **lower-bound hit** (“p at floor”), so we write `p <= 5e-06`.

## 6. Negative Results and Boundaries (Must Be Stated)

1. `gamma=0.8` is **not stable** as a frozen confirmatory window.
   - In `sis_runs=128` replications, the `no_switch` (and sometimes `switch`) signal at `N=30,keep=0.6,gamma=0.8` can collapse toward zero.
   - Therefore: `gamma=0.8` is explicitly excluded from the frozen confirmatory statement.

2. `drift`-only variant is **not eligible** for freezing.
   - Stratified tables show sign flips (some strata significant but negative), so it cannot serve as a monotone stability variable.

3. `no_switch` can be **borderline** at `sis_runs=64`.
   - One confirmatory replication at `sis_runs=64` yielded near-threshold p around 0.05 for `no_switch` at `gamma=0.2`.
   - Therefore: the frozen claim prioritizes `sis_runs=128` as the primary confirmatory evidence.

4. The claim is a **window claim**, not a global theorem.
   - We freeze a single parameter window (and the CG operator); we do not claim “D holds for all N/keep/gamma”.

5. N=52 requires a denoising confirmatory protocol.
   - Under the “standard” confirmatory settings at N=52, the `no_switch` and even `switch` variants can fluctuate across replications.
   - A dedicated denoise run (larger `samples_per_family` and `repeats`) restores strong significance at N=52 for the same stratum, indicating the instability can be variance-driven rather than mechanistic collapse.

## 6.1 Cross-Prediction Minimal Mechanism Check (C → D)

Since Prediction C identifies layer depth (`layer_count`, `mean_layer_gap`) as the primary mechanism behind entropy suppression, we ran a minimal cross-check on the frozen D window:

- Augment Tier-3 blocks with per-family `layer_count` and `mean_layer_gap` computed by reconstructing each original poset from `cg_original_samples.csv` seeds.
- Recompute Tier-3 using a partial Spearman statistic controlling for (`layer_count`, `mean_layer_gap`) within each block; keep the same within-block permutation null.

Result: in the frozen window (`N=30, keep_ratio=0.6, gamma=0.2`), the Tier-3 association remains strongly positive and significant after controlling for layer depth, indicating that `I_cg → improve_rank` is not merely a relabeling of the layer-depth mechanism.
